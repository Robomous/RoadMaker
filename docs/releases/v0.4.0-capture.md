# v0.4.0 README GIF — capture instructions (blocked-maintainer)

*The sprint requires a README GIF of the T-junction workflow. Capturing it
needs a real GL surface and a screen recorder; this environment is headless
(offscreen QPA renders no stable framebuffer for `grabFramebuffer` on this
platform), so per the sprint rule the exact instructions are recorded here
instead of silently skipping. One manual run, ~3 minutes.*

## Steps (macOS)

1. Build and launch the editor:
   `cmake --build --preset dev-macos --target roadmaker-editor && open build/dev-macos/editor/roadmaker-editor.app`
2. Start a screen recording of the editor window
   (`Cmd+Shift+5` → "Record Selected Portion", or QuickTime).
3. Perform, unhurried (this exact sequence reads well at GIF speed):
   a. Create Road (rural template): a gently curved road, 4 waypoints,
      left to right across the viewport.
   b. Create Road again: a short road approaching the first from below,
      ending ~10 m short of it.
   c. Create Junction tool: click the short road's free end (marker
      appears), then click the long road's body — the side-snap indicator
      lands on its reference line — press **Enter**. The junction forms.
   d. Press **Ctrl+Z** (everything vanishes), **Ctrl+Shift+Z** (it
      returns) — the one-undo-step story is the money shot.
4. Stop recording; convert to GIF ≤ 8 MB:
   `ffmpeg -i capture.mov -vf "fps=12,scale=960:-1" -loop 0 docs/img/t_junction.gif`
   (create `docs/img/` if missing; `brew install ffmpeg` if needed).
5. Reference it in README.md under the feature list:
   `![T-junction side attach](docs/img/t_junction.gif)` and commit both.
