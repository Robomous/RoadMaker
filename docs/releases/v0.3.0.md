# RoadMaker v0.3.0 — "Editor"

**M2 milestone complete.** RoadMaker crosses from a read-only viewer into an
interactive road-network **editor**: you can now author and edit a full lane-level
network entirely inside `roadmaker-editor`, with undo/redo on every change.
Junctions get real watertight 3D surfaces, and networks export to **OpenUSD**
alongside glTF and OpenDRIVE.

> _[GIF: author a two-road network → add a junction → 3D blended surface → export]_

Three platforms (macOS, Linux, Windows), CI-green with ASan+UBSan sanitizers.
Prebuilt editor packages (DMG / NSIS installer / AppImage) and Python wheels are
attached to the [GitHub release](https://github.com/Robomous/RoadMaker/releases/tag/v0.3.0).

---

## ✨ Highlights

### Interactive editing — the headline of M2
Everything is a kernel-level command on a single undo stack (`Ctrl+Z`/`Ctrl+Y`);
mid-drag previews never leak into history, and undo/redo round-trips are
byte-identical against the OpenDRIVE writer.

- **Select / Move** — multi-pick, rubber-band selection, direct node dragging.
- **Create Road** — draw clothoid-based roads with lane-profile templates and an
  endpoint-locked continuous-curvature (G1) fit that snaps to existing road ends.
- **Edit Nodes** — insert, delete, and drag reference-line nodes.
- **Lane Profile editor** — add/remove lanes, edit lane type, width, and road-mark.
- **Elevation editor** — set per-node height; the road's elevation profile re-fits
  as a smooth C1 cubic curve.
- **Create Junction** — pick 2+ road ends and auto-generate connecting roads with
  correct lane links; junctions regenerate deterministically when an incoming road
  is edited.
- **Delete** — removes a road/junction and its full referential closure
  (connections, connecting roads, links) in one undoable step.
- **Editable Properties panel** — rename roads inline.
- **New / Save** — the `.xodr` **is** the project file (authoring waypoints and
  junction arms persist via `<userData>`); the writer targets a selectable
  OpenDRIVE version (1.8.1 by default).

> _[GIF: lane profile editing + elevation curve]_

### Watertight 3D junction surfaces
Junction footprints are now real blended 3D surfaces with harmonic elevation that
respects the grade of every incoming road — replacing the M1 plan-view floors.
Surfaces export to OpenDRIVE ≥1.8 as `<planView>` + `<elevationGrid>`.

> _[GIF: before/after — flat floor vs. blended junction surface]_

### OpenUSD export
Export the tessellated network to **OpenUSD ASCII (`.usda`)** — meshes with
`UsdPreviewSurface` materials matching the glTF names, a
Road → LaneSection → Lane hierarchy, Y-up / meters. Validated against the OpenUSD
reference compliance checker in CI. Reads in usdview, Omniverse / Isaac Sim, and
Blender.

> _[GIF or screenshot: exported .usda opened in usdview]_

## 🧰 Kernel & standards

- **Command layer with undo/redo** in the kernel (`roadmaker::edit`), exposed to
  Python/headless for parity with the editor.
- **Incremental re-mesh** — only dirty roads (and junctions touching them)
  regenerate after an edit.
- **Kernel snapping queries** — endpoint / tangent-continuation / grid snapping,
  headless and testable.
- **Version-explicit OpenDRIVE writer** — emit 1.8.1 (needed for junction boundary
  / elevation grid) with per-write target selection.
- **Diagnostics carry ASAM rule IDs** and entity IDs
  (`asam.net:xodr:<ver>:<rule>`), so validator output cites the normative rule.

## 🐍 Python

New bindings + runnable examples land with their kernel features:
`fit_elevation_profile`, the junction generator, and `export_usda`
(feature-detected; wheels ship with USD **off**). `pip install roadmaker`.

## ⚠️ Known limitations (deferred to M3)

- **OpenUSD is `.usda` (ASCII) only** — no `.usdc`/`.usdz` crate writer (the
  chosen backend has no crate serializer; every major consumer reads `.usda`).
  Python wheels ship with USD disabled.
- **Lane widths only** — no `<border>` support; lane width is constant per lane
  (no polynomial width editing yet).
- **Road marks** render as a single strip (multi-line solid-solid comes in M3).
- **No superelevation** editing or lateral-frame tilt.
- **No sign/prop 3D assets** yet.
- **Junctions from foreign `.xodr` files** load without a recorded arm list and
  cannot be regenerated until recreated in-editor; changing an arm's lane count
  requires recreating the junction rather than an in-place regenerate.
