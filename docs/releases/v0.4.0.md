# RoadMaker v0.4.0 — the hardening release

*Draft release notes. The maintainer publishes after the
[v0.4.0 gate](../roadmap/golden_workflows/gate-v0.4.0.md) passes; the tag is
NOT created by automation.*

## Why this release looks different

After v0.3.0 shipped, the maintainer sat down and tried to build a real
map. Three things went wrong: **a T-intersection could not be drawn** (the
second thing anyone tries), **vertical design could not express a hill or
an overpass**, and **the editor crashed during normal use**. None of this
was caught by the milestone plans or the golden-scene acceptance — the spec
process covered the standard's elements, not the user's path.

v0.4.0 fixes those three things and changes the process so this class of
gap is measured from now on: every milestone gate now runs
[golden workflows](../roadmap/golden_workflows/README.md) — scripted user
paths with a time budget and a zero-crash requirement, executed by hand —
alongside the golden scenes. M3a shifts to v0.5.0.

## Highlights

- **T-junctions.** Select a road end, click another road's side, press
  Enter: the target splits, the junction area forms, and connecting roads
  for every legal turn appear — one undo step returns everything. The same
  main road can be teed repeatedly.
- **Vertical profile editor.** A Profile panel plots z(s) for the selected
  road with draggable elevation nodes and grade handles; a configurable
  advisory warns past 12 % grade.
- **Overpasses.** Cross Over / Cross Under re-profile a road to clear
  crossing roads by a configurable clearance (default 5.5 m) — grade
  separation without a junction.
- **Crash capture.** If the editor crashes, the next launch points at a
  local crash report (stack trace + the command log tail). Nothing is
  uploaded — reports stay on your machine; attaching one to a GitHub issue
  is always your choice (there is a `crash` template ready).
- **Soak-tested.** A seeded-random driver now hammers the real editing
  stack in CI under AddressSanitizer on every change. Its first runs found
  four real bugs (#87–#90), including two library exceptions that could
  kill the editor; all are fixed with pinned regressions.
- **Autosave, hardened.** A recovery copy is written right before every
  junction regeneration, and autosave can be disabled from the File menu.

Plus the early-M3a kernel groundwork that had already landed on `main`:
OpenDRIVE `<objects>`/`<signals>`, road-mark completions, generated-junction
`<boundary>`, the esmini CI round-trip gate, and the first user guide.

## Known limits (honest edges)

- Turn permissions at T-junctions are not editable yet — every legal turn
  is generated; pruning is scheduled after the sprint.
- Superelevation editing remains M3a+ scope; the Profile panel edits
  elevation only.
- Starting a NEW road on another road's side (tee-on-commit from Create
  Road) and grade-matched creation are tracked as #95 / #97.
- "Mountain-type" terrain is a pending maintainer decision
  ([ADR-0006](../decisions/0006-terrain-scope.md), issue #83).

## Gate

Release requires the maintainer to pass GW-1 ("first network": road → tee →
overpass → lane edit → junction drag → undo/redo ×10 → save/reload →
glTF/USD/esmini, ≤ 20 min, zero crashes) and GW-2 (SIGKILL → autosave
recovery, ≤ 2 min lost), with a 60-minute ASan soak clean.
